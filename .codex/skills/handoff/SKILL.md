# /handoff

When the user invokes `/handoff`, run this flow.

## 1. Gather inputs

Required:

- **title** — short imperative summary of what this snapshot captures (e.g., `Mid-execute snapshot before compact`)
- **topic** — free-form short identifier; slugified for the filename (e.g., `sprint-3-mid-execute`)
- **author** — typically the calling member's `@id` in parley; the human operator otherwise

Optional:

- **trigger** — `manual` (default) | `pre_compact` (reserved for the Sprint 6 PreCompact hook) | `session_end`
- **sprint_id** + **stage** — must be paired (both set or both null). `stage` ∈ `plan | execute | retro`. Set when there's an active sprint at handoff time.
- **since-handoff-id** — id of the prior handoff (cursor for "since last handoff"). When omitted, the body's "Since last handoff" section becomes a pointer to sibling indices rather than a delta.
- **since-msg-id** — parley message cursor. **Caller's responsibility to populate** (the helper lib stays parley-agnostic). See §2 for the shell-out flow.
- **body** — inline markdown body. Takes precedence over `--body-from-file`.
- **body-from-file** — path to a file whose content becomes the handoff body. Frontmatter in the source file is stripped.
- **linked-decisions / linked-issues / linked-tasks / linked-msg-ids** — comma-separated ids relevant to this handoff window.
- **next-action** — one-sentence resume hint for the next reader.

If body and body-from-file are both omitted, the helper auto-generates a minimal body from sprint state (sibling INDEXes are gracefully skipped if absent).

If any required input is missing or ambiguous, use `AskUserQuestion` to collect.

## 2. Determine author + parley context (parley-coupling lives HERE, not in the lib)

If in a parley session:

1. Run `parley whoami` to discover the calling member's id. Use `@<id>` for `--author`.
2. To populate `--since-msg-id` with the current parley message cursor, shell out:
   ```bash
   parley get --limit 1 --json | tail -1
   ```
   Parse the JSON line and use its `msg-id` value for `--since-msg-id`.
3. Capture relevant `--linked-msg-ids` (parley msg-IDs that occurred during this handoff window) for the durable provenance chain.

If NOT in a parley session, leave `--since-msg-id` and `--linked-msg-ids` unset — both default to None / empty.

The helper lib (`record_handoff`) is parley-agnostic by design (CLAUDE.md Hard Rule 5). The parley shell-outs above run at the SKILL layer so the lib stays clean.

## 3. Write the handoff

From the repo root, invoke the CLI:

```bash
.venv/bin/python3 .claude/scripts/dev-mgmt/cli.py handoff \
    --title "<title>" \
    --topic "<topic>" \
    --author "@<member>" \
    [--trigger manual|pre_compact|session_end] \
    [--sprint-id "<sprint-id>" --stage plan|execute|retro] \
    [--since-handoff-id "<prior-id>"] \
    [--since-msg-id "<msg-id>"] \
    [--body "<inline body>" | --body-from-file "<path>"] \
    [--linked-decisions "<csv>"] \
    [--linked-issues "<csv>"] \
    [--linked-tasks "<csv>"] \
    [--linked-msg-ids "<csv>"] \
    [--next-action "<one-sentence>"]
```

If the repo doesn't have a `.venv/`, fall back to `python3` (PyYAML must be available).

The CLI:
- auto-generates the id (`YYYY-MM-DD-HHMM-<slugified-topic>`)
- validates the frontmatter against the D6.A Handoff schema
- writes `docs/handoffs/<id>.md`
- atomically re-renders `docs/handoffs/INDEX.md`
- prints the written path on stdout

A `ValidationError` exits non-zero and writes nothing. A common cause is passing `--sprint-id` without `--stage` (they must be paired).

## 4. Report back

Tell the user:

- the path to the written handoff
- the auto-generated handoff id (so future `--since-handoff-id` callers can reference it)
- a one-line summary of what was captured (active sprint at handoff time, trigger, link counts)

If `trigger=pre_compact`, note that this handoff was machine-generated by the PreCompact hook (Sprint 6 deliverable) and may want a human follow-up.

## Notes

- **Triggers**: `manual` is the default for human-invoked handoffs; `pre_compact` is reserved for the Sprint 6 PreCompact hook that auto-calls `/handoff` before CC compaction; `session_end` is for explicit session-end snapshots (manual or future hook).
- **ID uniqueness**: `YYYY-MM-DD-HHMM-<slug>` has a 1-UTC-minute collision window if two handoffs of the same slug fire concurrently. If collision is observed in practice, the lib will widen to `HHMMSS` or add a per-minute NN counter.
- **Cross-link semantics**: the `linked_*` arrays are opaque string lists in Sprint 3 — content validation against existing entity ids is deferred to Sprint 7's `cross_links.py`.
- **Workshop importability**: every field maps cleanly to a Workshop entity column (per CLAUDE.md Hard Rule 6). D6.A fills the §6 design-doc gap; the design doc will be amended during session-end ritual.
- **Parley-agnostic base**: this skill works in a solo CC session in a fresh repo too, as long as `docs/handoffs/` (or its parents) can be written. Parley-coupling lives in §2's shell-outs, NOT in the helper lib.
